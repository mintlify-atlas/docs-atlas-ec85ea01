FixTweetBot uses [Masonite ORM](https://orm.masoniteproject.com/) for database management. This page documents the core database models and their fields.

## Guild

Represents a Discord server (guild) and its bot configuration.

**Location:** `database/models/Guild.py`

**Table:** `guilds`

### Fields

<ParamField path="id" type="bigint" required>
  Discord guild ID (unique identifier)
</ParamField>

<ParamField path="keywords" type="json" default='["fxignore"]'>
  List of keywords to filter messages (e.g., "fxignore")
</ParamField>

<ParamField path="keywords_use_allow_list" type="boolean" default="false">
  Whether to use allow list for keywords filtering
</ParamField>

<ParamField path="text_channels_use_allow_list" type="boolean" default="false">
  Whether to use allow list for text channels
</ParamField>

<ParamField path="members_use_allow_list" type="boolean" default="false">
  Whether to use allow list for members
</ParamField>

<ParamField path="roles_use_allow_list" type="boolean" default="false">
  Whether to use allow list for roles
</ParamField>

<ParamField path="roles_use_any_rule" type="boolean" default="false">
  Whether to apply any role rule matching
</ParamField>

<ParamField path="reply_to_message" type="boolean" default="false">
  Whether to reply to the original message
</ParamField>

<ParamField path="reply_silently" type="boolean" default="false">
  Whether replies should be silent (no notification)
</ParamField>

<ParamField path="webhooks" type="boolean" default="false">
  Whether to respond to webhooks (premium feature)
</ParamField>

<ParamField path="original_message" type="enum" default="remove_embeds">
  How to handle the original message:
  - `nothing` - Leave the original message unchanged
  - `remove_embeds` - Suppress embeds on the original message
  - `delete` - Delete the original message
</ParamField>

<Expandable title="View Settings">
  <ParamField path="twitter_view" type="enum" default="normal">
    Twitter embed view mode:
    - `normal` - Standard Twitter embed
    - `gallery` - Gallery view for multiple images
    - `text_only` - Text-only view
    - `direct_media` - Direct media embed
  </ParamField>

  <ParamField path="instagram_view" type="enum" default="normal">
    Instagram embed view mode:
    - `normal` - Standard Instagram embed
    - `direct_media` - Direct media embed
    - `gallery` - Gallery view for multiple images
  </ParamField>

  <ParamField path="tiktok_view" type="enum" default="normal">
    TikTok embed view mode:
    - `normal` - Standard TikTok embed
    - `gallery` - Gallery view
    - `direct_media` - Direct media embed
  </ParamField>

  <ParamField path="bluesky_view" type="enum" default="normal">
    Bluesky embed view mode:
    - `normal` - Standard Bluesky embed
    - `direct_media` - Direct media embed
    - `gallery` - Gallery view
  </ParamField>

  <ParamField path="snapchat_view" type="enum" default="normal">
    Snapchat embed view mode (via EmbedEZ):
    - `normal` - Standard embed
    - `direct_media` - Direct media embed
  </ParamField>

  <ParamField path="ifunny_view" type="enum" default="normal">
    iFunny embed view mode (via EmbedEZ)
  </ParamField>

  <ParamField path="imgur_view" type="enum" default="normal">
    Imgur embed view mode (via EmbedEZ)
  </ParamField>

  <ParamField path="weibo_view" type="enum" default="normal">
    Weibo embed view mode (via EmbedEZ)
  </ParamField>

  <ParamField path="imageboards_view" type="enum" default="normal">
    Imageboards embed view mode (via EmbedEZ)
  </ParamField>

  <ParamField path="pinterest_view" type="enum" default="normal">
    Pinterest embed view mode (via EmbedEZ)
  </ParamField>
</Expandable>

<ParamField path="created_at" type="timestamp">
  Record creation timestamp
</ParamField>

<ParamField path="updated_at" type="timestamp">
  Record last update timestamp
</ParamField>

### Relationships

## Member

Represents a Discord guild member and their filtering settings.

**Location:** `database/models/Member.py`

**Table:** `members`

**Extends:** `AFilterModel`

### Fields

<ParamField path="id" type="integer" required>
  Auto-incrementing primary key
</ParamField>

<ParamField path="user_id" type="bigint" required>
  Discord user ID (indexed)
</ParamField>

<ParamField path="guild_id" type="bigint" required>
  Reference to the parent guild ID
</ParamField>

<ParamField path="bot" type="boolean">
  Whether the member is a bot (bots are on deny list by default)
</ParamField>

<ParamField path="on_deny_list" type="boolean" default="false">
  Whether the member is on the deny list
</ParamField>

<ParamField path="on_allow_list" type="boolean" default="false">
  Whether the member is on the allow list
</ParamField>

<ParamField path="created_at" type="timestamp">
  Record creation timestamp
</ParamField>

<ParamField path="updated_at" type="timestamp">
  Record last update timestamp
</ParamField>

### Relationships

<ResponseField name="guild" type="BelongsTo">
  Parent Guild model
</ResponseField>

### Methods

```python
Member.find_or_create(d_member: discore.Member, guild: Guild | None = None, 
                      guild_kwargs: dict | None = None, **kwargs) -> Member
```

Finds existing member by user_id and guild_id, or creates a new one. Bots are automatically added to deny list.

```python
Member.find_get_enabled(d_member: discore.Member, guild: Guild | None = None) -> bool
```

Checks if a member is enabled. Bots are disabled by default.

```python
Member.reset_lists(guild: Guild) -> None
```

Resets deny and allow lists for all members in a guild. Bots remain on deny list.

**Table:** `events`

### Fields

  Event name/type identifier
</ParamField>

<ParamField path="data" type="json" default="{}">
  Additional event data in JSON format
</ParamField>

<ParamField path="created_at" type="timestamp">
  Event timestamp
</ParamField>

<ParamField path="updated_at" type="timestamp">
  Record last update timestamp
</ParamField>

### Methods

```python
Event.since(event_name: str | None = None, days: int = 0, 
            hours: int = 0, minutes: int = 0, seconds: int = 0) -> list[Event]
```

Returns events since a specified time period, optionally filtered by event name.

```python
await Event.buff_cr(event: dict) -> None
```

Buffers event creation and flushes to database every 5 seconds for performance.

<Note>
  Event tracking is only active when `discore.config.analytic` is enabled.
</Note>

  
  Checks if the entity is enabled based on allow/deny list configuration.

  - If guild uses deny list: returns NOT `on_deny_list` status
