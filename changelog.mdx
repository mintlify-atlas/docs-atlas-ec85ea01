## What's Included in Releases

Each release typically includes:

### New Features

Major additions and enhancements to the bot's functionality, such as:
- New platform support
- Additional embed view options
- Enhanced filtering capabilities
- Premium features
- Translation improvements

### Bug Fixes

Resolutions to reported issues and bugs:
- Link detection improvements
- Embed generation fixes
- Performance optimizations
- Configuration issue resolutions

### Updates

Improvements to existing features:
- Better error handling
- UI/UX enhancements in settings
- Updated dependencies
- Security patches
- Documentation updates

### Breaking Changes

Occasionally, releases may include breaking changes:
- Configuration format updates
- Database migration requirements
- API changes for self-hosters
- Deprecated feature removals

<Warning>
  Breaking changes are clearly marked in release notes. Always read the release notes before updating a self-hosted instance.
</Warning>

## Staying Updated

### For Public Bot Users

If you're using the public FixTweetBot instance:
- Updates are applied automatically
- No action required on your part
- Changes take effect immediately
- Check `/about` to see the current version

### For Self-Hosters

If you're self-hosting FixTweetBot:

<Steps>
  <Step title="Check for Updates">
    Monitor the [GitHub Releases](https://github.com/Kyrela/FixTweetBot/releases) page or watch the repository
  </Step>
  <Step title="Review Release Notes">
    Read the changelog for breaking changes and migration steps
  </Step>
  <Step title="Backup Your Data">
    Always backup your database before updating
  </Step>
  <Step title="Update Your Instance">
    Follow the update instructions for your deployment method:
    
    **Docker:**
    ```bash
    docker compose pull
    docker compose up -d
    ```
    
    **Bare Metal:**
    ```bash
    git pull
    pip install -r requirements.txt
    masonite-orm migrate -C database/config.py -d database/migrations
    ```
  </Step>
  <Step title="Verify the Update">
    Check logs for any errors and verify the bot is functioning correctly
  </Step>
</Steps>

## Version Numbering

FixTweetBot follows semantic versioning (SemVer):

- **Major version** (X.0.0): Breaking changes, major rewrites
- **Minor version** (3.X.0): New features, backwards-compatible
- **Patch version** (3.3.X): Bug fixes, minor improvements
